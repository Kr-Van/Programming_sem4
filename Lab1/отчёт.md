## **Описание работы компонентов программы**

В этой работе для фильтрации чисел Фибоначчи используется математическое свойство. Функция `is_sqrt(n)` проверяет, является ли число полным квадратом. На её основе строится проверка: число принадлежит ряду Фибоначчи тогда и только тогда, когда хотя бы одно из выражений \((5n^2 + 4)\) или \((5n^2 - 4)\) является полным квадратом. Перед вычислениями обязательно проверяется корректность входного диапазона (исключаются отрицательные числа под корнем).
<br> <br>
Для генерации последовательности разработана сопрограмма `fib_gen()`. Она демонстрирует принцип кооперативной многозадачности и работает в бесконечном цикле `while True`, принимая команды через метод `.send(num)`. Для автоматического перевода сопрограммы в состояние ожидания (прайминга) применяется декоратор `start(g)` на базе `functools.wraps`. Если на вход передаются невалидные типы, пустые значения или `0`, сопрограмма корректно возвращает `None`, в остальных случаях — генерирует список из первых чисел Фибоначчи.
<br> <br>
Класс `FibonacciIter` реализует классический протокол итератора (методы `__iter__` и `__next__`). Особенность данного итератора заключается в "ленивой" фильтрации — элементы проверяются по мере запроса, что экономит память при работе с большим объёмом данных. Если элемент коллекции отрицательный, итератор пропускает его с помощью оператора `continue`. Если элемент удовлетворяет условию Фибоначчи, индекс смещается, а число возвращается внешнему коду. При выходе за границы коллекции возбуждается исключение `StopIteration`.
```python
elif is_sqrt(5 * n**2 + 4) or is_sqrt(5 * n**2 - 4):
    self.idx += 1
    return n
```
Класс `SimpleFibonacciIter` реализует альтернативный подход к обработке данных с помощью метода `__getitem__`. При инициализации объекта список полностью фильтруется один раз с помощью спискового включения (List Comprehension). Это обеспечивает быстрый произвольный доступ по индексу, но требует полной загрузки данных в память. Метод `__getitem__(idx)` осуществляет доступ к элементам отфильтрованного массива, а в случае запроса несуществующего индекса стандартно генерирует исключение `IndexError`.
```python
def __getitem__(self, idx):
    try:
        return self.data[idx]
    except IndexError:
        raise IndexError
```

## **Принципиальные отличия подходов**

### **Итераторы vs Сопрограммы**
- **Итераторы** предоставляют односторонний поток данных от источника к потребителю.
- **Сопрограммы** организуют двустороннюю коммуникацию, позволяя внешнему коду динамически влиять на внутреннее состояние через `.send()`.

### **Два типа итераторов**
- **Ленивая обработка (`FibonacciIter`)** обеспечивает экономию оперативной памяти при работе с большими данными, но может приводить к более медленному поэлементному доступу.
- **Предварительная фильтрация (`SimpleFibonacciIter`)** гарантирует мгновенный произвольный доступ по индексу, но требует полной загрузки коллекции в память при создании объекта.
