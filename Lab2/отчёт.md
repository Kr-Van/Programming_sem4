## **Описание работы компонентов программы**

В этой работе для динамического расширения возможностей базового класса без изменения его исходного кода используется структурный шаблон проектирования «Декоратор». Основной абстрактный интерфейс `CurrenciesProvider` и класс обёртки `CurrenciesDecorator` спроектированы наиболее строгим образом на базе абстрактных базовых классов (`abc.ABC` и декоратора `@abstractmethod`).
<br> <br>
Роль конкретного компонента выполняет класс `CurrenciesProviderJSON`. Он отвечает за чистую бизнес-логику: выполняет HTTP-запрос к официальному API Центрального банка РФ, проверяет статус ответа и возвращает исходные данные в виде словаря. Метод `save_to_file` сохраняет полученную информацию в JSON-файл с кодировкой UTF-8.
<br> <br>
Для изменения структуры вывода реализованы два конкретных декоратора:
1. **`CSVDecorator`**: Извлекает вложенный словарь `"Valute"` из JSON-ответа компонента, использует потоковый буфер `io.StringIO` и встроенную библиотеку `csv` (класс `csv.DictWriter`) для преобразования объектов в плоскую таблицу.
```python
writer = csv.DictWriter(
    output,
    fieldnames=["CharCode", "Name", "Nominal", "Value", "Previous"]
)
writer.writeheader()
for info in valutes.values():
    writer.writerow({
        "CharCode": info["CharCode"],
        "Name": info["Name"],
        "Nominal": info["Nominal"],
        "Value": info["Value"],
        "Previous": info.get("Previous")
    })
```
2. **`YAMLDecorator`**: Использует внешнюю библиотеку `PyYAML` для сериализации структурированного словаря в формат YAML. Метод `save_to_file` считывает текстовое представление и записывает его на диск в текстовый файл.
```python
def save_to_file(self, filename: str) -> None:
    data = self.get_currencies()
    with open(filename, "w", encoding="utf-8") as file:
        file.write(data["yaml_text"])
```

## **Принципиальные отличия подходов и специфика DAST/DATS**

### **Паттерн Декоратор vs Наследование**
- **Наследование** создаёт статическую связь на этапе компиляции/интерпретации, привязывая логику к конкретным классам.
- **Декораторы** позволяют комбинировать форматы динамически в рантайме. Например, можно обернуть один декоратор в другой (слоёная структура), сохраняя при этом единый контракт вызовов `operation()`.

### **Особенности тестирования (Тесты и Спецификации)**
Программа полностью соответствует стандартам качества кода:
- **PEP 484 / PEP 257 / PEP 8**: Код содержит строгие аннотации типов (включая сложные генерики `Dict[str, Any]`), подробную документацию методов (Docstrings).
- **Изолированные тесты**: Тестирование не зависит от доступности интернета или серверов ЦБ РФ, так как реальный сетевой запрос подменяется объектами `Mock()` с помощью патчей `@patch("requests.get")`. Каждый класс проверяется как на корректность структуры данных на выходе, так и на успешность файловых операций ввода-вывода (с последующей автоматической очисткой диска `os.remove`).
