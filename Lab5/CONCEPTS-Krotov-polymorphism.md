# Полиморфизм в ООП

**Полиморфизм** — это один из ключевых принципов объектно-ориентированного программирования, который позволяет объектам с разной внутренней реализацией иметь одинаковый интерфейс взаимодействия (одинаковые имена методов). Простыми словами, полиморфизм позволяет писать универсальный код, который может работать с разными типами объектов, не вдаваясь в детали их конкретных классов, при условии, что они поддерживают общее поведение.

В приведённом примере функция `activate_security_system` демонстрирует полиморфизм на практике. Она принимает список различных защитных устройств — `Camera` (камера), `MotionSensor` (датчик движения) и `SmartLock` (умный замок). Каждый из этих классов имеет свою уникальную внутреннюю логику и атрибуты, однако все они реализуют единый метод `trigger()`. Функция обрабатывает их единообразно в цикле, вызывая этот метод, что позволяет легко масштабировать систему безопасности новыми типами устройств без изменения логики самой функции.

### Пример кода

```python
from abc import ABC, abstractmethod


class SecurityDevice(ABC):
    """Базовый абстрактный класс для всех устройств системы безопасности."""

    @abstractmethod
    def trigger(self) -> None:
        """Абстрактный метод для активации тревоги или защитного режима."""
        pass


class Camera(SecurityDevice):
    """Класс охранной камеры со своей уникальной логикой."""

    def __init__(self, location: str):
        self.location = location

    def trigger(self) -> None:
        """Реализация метода: запуск записи видеопотока."""
        print(f"[КАМЕРА]: Обнаружено движение в локации '{self.location}'. Включена запись Full-HD.")


class MotionSensor(SecurityDevice):
    """Класс инфракрасного датчика движения."""

    def __init__(self, sensitivity: int):
        self.sensitivity = sensitivity

    def trigger(self) -> None:
        """Реализация метода: включение сирены при нарушении периметра."""
        print(f"[ДАТЧИК]: Зафиксировано тепловое излучение (чувствительность {self.sensitivity}%). Включена сирена!")


class SmartLock(SecurityDevice):
    """Класс электронного умного замка."""

    def __init__(self, door_id: int):
        self.door_id = door_id

    def trigger(self) -> None:
        """Реализация метода: блокировка дверей."""
        print(f"[ЗАМОК]: Дверь №{self.door_id} переведена в режим полной экстренной блокировки.")


def activate_security_system(devices: list[SecurityDevice]) -> None:
    """Универсальная функция, демонстрирующая полиморфное поведение.

    Принимает список любых охранных устройств и активирует их.
    """
    print("--- АКТИВАЦИЯ ОБЩЕГО РЕЖИМА ТРЕВОГИ ---")
    for device in devices:
        # Полиморфный вызов: имя метода одинаковое, но поведение зависит от класса объекта
        device.trigger()


# --- Использование ---
if __name__ == "__main__":
    # Создаем объекты совершенно разных классов
    hall_camera = Camera("Главный холл")
    window_sensor = MotionSensor(sensitivity=85)
    main_gate_lock = SmartLock(door_id=101)

    # Помещаем их в единую коллекцию
    security_perimeter = [hall_camera, window_sensor, main_gate_lock]

    # Передаем в универсальную функцию обработки
    activate_security_system(security_perimeter)
```

### Ключевые особенности, которые добавляет Python

1. **Утиная типизация (Duck Typing)**: В отличие от строго типизированных языков (Java, C++), где для полиморфизма объекты *обязаны* наследоваться от одного класса (`SecurityDevice`) или имплементировать общий интерфейс, в Python это делать необязательно. Действует правило: *«Если это выглядит как утка, плавает как утка и крякает как утка, то это утка»*. Функция `activate_security_system` успешно сработает даже с объектом стороннего класса, который никак не связан с `SecurityDevice`, если внутри него просто объявлен метод `trigger()`.
2. **Отсутствие статического перегруза методов (Overloading)**: В Python полиморфизм работает исключительно на уровне объектов (динамический полиморфизм). Вы не можете создать внутри одного класса два метода с одинаковым именем, но разными типами аргументов (например, `def process(x: int)` и `def process(x: str)`). Новый метод всегда заменяет старый. Множественное поведение параметров при необходимости реализуется динамической проверкой типов прямо внутри тела метода.
